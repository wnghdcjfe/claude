# MD2Short 문서 인덱스

이 파일이 `docs/` 문서 목록의 단일 진실원(SSOT)이다. 새 문서를 추가하거나 문서 역할을 바꾸면 이 파일을 먼저 갱신하고, `AGENTS.md`에는 핵심 진입점 링크만 유지한다.

| 파일 | 범위 | 언제 읽나 |
| --- | --- | --- |
| [`../AGENTS.md`](../AGENTS.md) | 에이전트 작업 원칙, 레이어 경계, 코딩 컨벤션 요약 | 항상 |
| [`../README.md`](../README.md) | 설치, 실행, 테스트, CLI 사용법 | 환경 셋업·재현 시 |
| [`PRD.md`](PRD.md) | 제품 요구사항(WHAT/WHY), 목표·비목표, 사용자 시나리오 | 새 기능 설계·제품 판단 시 |
| [`architecture.md`](architecture.md) | 시스템 설계(HOW), 파이프라인, 상태 머신, 데이터 모델, 산출물 구조 | 구현·통합·디버깅 시 |
| [`voice.md`](voice.md) | 음성/voice 설정과 TTS 운용 기준 | 음성 품질·voice 변경 시 |
| [`memory.md`](memory.md) | Memory 시스템 규칙 명세(스키마, 승격 기준, drift 감사) | 실패 기록·승격·감사 시 |
| [`commit.md`](commit.md) | 커밋 메시지 양식과 trailer 규칙 | 커밋·푸시 직전 |

## 갱신 규칙

- 문서 추가/삭제/역할 변경 시 이 인덱스를 같은 변경에 포함한다.
- `AGENTS.md`에는 전체 목록을 중복하지 않고, 작업자가 반드시 알아야 하는 핵심 링크만 둔다.
- `docs/memory.md`는 기록 규칙이고, 실제 실패 데이터는 `memory/` 아래에 저장한다.
