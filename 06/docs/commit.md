# 커밋 메시지 규칙

상위 진입점: `AGENTS.md` Lore Commit Protocol.

커밋은 사용자가 명시한 경우에만 수행함. 커밋 메시지는 diff 요약보다 **왜 이 변경을 했는지**를 먼저 남기는 결정 기록이어야 함.

---

## 기본 형식

첫 줄은 `<type>(<scope>): <intent>`. body와 메타 trailer는 모두 마크다운 불릿으로 작성함.

```markdown
<type>(<scope>): <intent: 변경 이유를 설명하는 한 줄>

- <body 불릿: 제약·접근 방식·영향 범위>
- <필요 시 추가>

- **제약**: <외부/프로젝트 제약>
- **기각**: <검토했으나 채택 안 한 대안> | <사유>
- **확신도**: 낮음 | 중간 | 높음
- **영향범위**: 좁음 | 중간 | 넓음
- **복구난이도**: 깔끔 | 지저분 | 불가역
- **당부**: <후속 수정자에게 남길 주의점>
- **검증**: <실행한 검증>
- **미검증**: <알려진 검증 공백>
- **관련**: <관련 이슈/커밋/문서>
```

body 리스트와 메타 리스트는 빈 줄로 분리해서 별도 섹션처럼 보이게 함. 한글 키는 어차피 `git interpret-trailers`의 토큰 규칙(`[A-Za-z0-9-]+`)을 통과 못 하므로 불릿으로 둬도 자동화 손실 없음. type prefix 자동화는 첫 줄만 보므로 영향 없음.

---

## type

| type | 의미 |
| --- | --- |
| `feat` | 새 기능 추가 |
| `fix` | 버그 수정 |
| `refactor` | 동작 변경 없는 코드 정리 |
| `perf` | 성능 개선 |
| `docs` | 문서 변경 |
| `test` | 테스트 추가/수정 |
| `chore` | 의존성·설정·빌드 잡일 |
| `ci` | CI 설정 변경 |
| `style` | 포맷·세미콜론 등 의미 없는 변경 |
| `revert` | 이전 커밋 되돌림 |

`<scope>`는 선택. 변경 영역(예: `timing`, `cli`, `numeric`)을 짧게 적음.

---

## 필수 규칙

1. 첫 줄은 `<type>(<scope>): <intent>` 형식. type은 위 표에서 고름.
2. intent는 **why 중심**으로 한 줄에 적음. 한국어 `~함` 톤.
3. body · 메타 trailer 모두 마크다운 불릿(`- 산문` 또는 `- **키**: 값`) 형식으로 적음.
4. body 리스트와 메타 리스트는 빈 줄로 분리함.
5. `**검증**`에는 실제 실행한 검증을 적음. 실행 안 한 건 `**미검증**`에 솔직히 남김.
6. 외부 API, 렌더링, 비용 계산, 상태 전이 변경은 `**제약**`과 `**당부**`를 우선 남김.
7. 대안 검토가 있었다면 `**기각**`에 남겨 같은 논의를 반복하지 않게 함.

---

## 메타 trailer 의미

| 키 | 의미 |
| --- | --- |
| `제약` | 코드만 보면 보이지 않는 외부/프로젝트 제약 |
| `기각` | 검토했지만 채택하지 않은 대안과 사유 |
| `확신도` | 작성자의 확신 수준 (`낮음`, `중간`, `높음`) |
| `영향범위` | 변경의 영향 범위 (`좁음`, `중간`, `넓음`) |
| `복구난이도` | 되돌리기 난이도 (`깔끔`, `지저분`, `불가역`) |
| `당부` | 후속 수정자가 지켜야 할 경고/주의 |
| `검증` | 실제 수행한 검증 |
| `미검증` | 알려진 검증 공백 |
| `관련` | 관련 이슈, 커밋, 문서 |

---

## 예시

```markdown
refactor(timing): 나레이션 타이밍을 스크립트 추정 대신 Scribe 기준으로 유지함

- TTS 길이는 보이스 속도와 `ElevenLabs prosody`에 따라 달라짐
- 렌더 타이밍은 스크립트 글자수가 아니라 생성된 오디오 + `words.json`에서만 와야 함
- 공용 helper 바깥에서 별도 타이밍 추정을 두면 voice 설정 변경 시 drift가 다시 발생함

- **제약**: words.json이 타이밍의 single source of truth임
- **기각**: 한국어 글자수로 길이 추정 | 보이스 설정마다 drift 발생함
- **확신도**: 높음
- **영향범위**: 좁음
- **복구난이도**: 깔끔
- **당부**: 공용 helper 바깥에서 별도 타이밍 추정을 추가하지 말 것
- **검증**: uv run pytest tests/test_qc.py tests/test_render.py
- **미검증**: 프로덕션 네트워크 조건에서의 ElevenLabs latency
```
